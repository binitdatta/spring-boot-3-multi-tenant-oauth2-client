Monitoring Applications With Prometheus

  • Monitoring means keeping an eye on something, like Hospitals keep a close eye on every critical patient.
  • As all major business and govrenment organizations today are run by software and software can go from healthy to broken very quickly, it is critical that we're monitoring them closely as well.
  • What exactly can we or shall we monitor on these Applications
  • First, applications today are built using components like APIs, UIs, Mobile Apps, Databases, Message Brokers similar to a Car being made up of components like an Engine, transmission, four tires, breaks, windows etc.
  • Like a Car has a monitoring system and a dashboard that highlight if anything goes wrong (Engine Check Light), we monitor our REST APIs , UIs, Databases, Message Brokers and build Dashboards to highlight problems
  • Prometheus is one of those free, open source and extreamly popular monitoring tool known as a Time Series Database
  • A Time Series Database collects various metrics (aka numbers) against a flow of passing time (this second, the next, and the next and so on) and stores them in its Database
  • However, numbers by themselves are not very useful. If we say to a nurse a patient's heartbeat just increased, she would be confused about which patient we are talking about
  • That is why numbers need to be associated with labels that can qualify them further. If we say the patient on room / bed number 20 on second floor has his heart rate increased, the Nurse can attend to that.
  • Likewise, Prometheus can collect these metrics or performance number (requests served per second, average time taken to respond, disk utilized, disk free, 99th percentile and so on) from hundreds of VMs , Disks and Containers
  • Then Prometheus attaches labels to qualify the web server, database, the application, to identify the metrics better to be queried later
  • In addition, it stores these massive data in a distributed database providing safety against disk crashes and data losses
  • It also supports a basic Graphical UI to query and chart graphs on the result data while Grafana is known as a open source popular Graphical UI based on Prometheus
  • Each service monitored by Prometheus provides an API for Prometheus to call and collect metrics and these metadata is provided in a rules file to Prom.
  • Finally, Prometheus has a built-in AlertManager Prometheus can periodically evaluate alerting rules if they are breached and send notifications to interested parties through the Alert Manager.
