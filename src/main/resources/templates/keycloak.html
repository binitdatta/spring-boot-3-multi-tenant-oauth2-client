KeyCloak As an Open Source IAM Server

  • Business Organizations typically have various departments dedicated for different types of work
  • We have Sales, Customer Support, Production, R&D, Services and Payrol/Finance and many other departments in any business organization
  • These departments might employ hundreds or even thousands of employees
  • Irrespective of which department they work for, however, all employees have to get their access card to enter the office buildings from a Central single Security Department
  • The R&D Department does not have its own Access Card System that is different from the Services department
  • In terms of software, UI, APIs, entire Products, this is the same. There is a single central Authorization Server that authenticates and authorizes access sought by Clients, APIs calling Other APIs and users
  • Like in market segment, there is a lot of choices, the market for this Authorization Server has lots of choices
  • Commercial Identity and Access Management (IAM) Services like Okta, Azure Active Directory, AWS Cognito, Auth0 are there.
  • KeyCloak is a similar feature rich, open source IAM Server from Red Hat and IBM that is very popular and mature.
  • Like Supply Chain is a specialized software segment and companies want to use the best Supply Chain software out there, IAM Software is also a niche software segment that is critically important
  • Like Companies do not write home-grown supply chain solutions and delegate to product vendors, they also delegate the IAM tasks to KeyCloak
  • KeyCloak supports multiple external Identity Providers like Okta, ADP, Google, Github and AWS and also has support for native database users
  • In addition, KeyCloak endlessly customizable with Java Code to fulfill the needs that is not available out of the box.
