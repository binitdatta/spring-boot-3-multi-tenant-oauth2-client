OIDC Stands for Open ID Connectivity

  • The first word is Open which means anyone can support it free of cost
  • When OAuth was created by the Internet Engineering Task Force (IETF), it was all about authorization of third parties openly
  • OAuth and then OAuth2 have nothing to also support accessing user profile information like named, addresses, email, phone numbers etc.
  • What OAuth2 does for open authorization, OIDC does the same for open accessibility of user profile information
  • Third party Identity Providers like Okta, Google, Facebook, Amazon, Microsoft, IBM and others support both OAuth2 and OIDC
  • When your IdP support both OAuth2 and OIDC, you get user profile information included in your access token
  • Additionally, you get an end point called the /user-info endpoint, calling which you can get detailed user profile information uniformly
  • If you see that your IdP is supporting OIDC, that means it also does implicitly support OAuth2
