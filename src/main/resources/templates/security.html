Application Security is as important as The Physical Security of our Homes, Office Buildings

  • Properly Secured Applications Prevent Unauthorized Access by Malicious Users trying to steal data and Intellectual Properties (IP)
  • Attacks can take place to steal data, IP or simply to disrupt critical services in the form of Distributed Denial of Service
  • When hackers use a single computer to pursue their attacks, it is called Denial of Service.
  • When hackers use thousands of computers distributed across the world to pursue their attacks, it is called Denial of Service.
  • When malicious users invoke services meant for legitimate users , it cost resources CPU, Memory and Disk access, preventing the service from serving users.
  • Besides, causing malicious code to be executed as part of the application (CSRF or Cross Site Request Forgery), exploiting weaknesses to inject malicious SQL statements also cause damage
  • All modern stacks / development framework have strong tools and industry standard frameworks to protect against these type of attacks
  • It is thus of utmost importance to use these standard security frameworks, rather than using home-grown solutions to secure applications
  • In this application, we will demonstrate how to secure a UI and a Service API using the latest Spring Security Framework (Version 6.x).
